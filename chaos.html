Using chaos to send secret messages

Try it out here!

If you want to encode an image, upload it here and save the result. Then, you can send it to somebody and have them decode it!

If you want to decode a masked image, upload it here.

If you want to decode a masked image, but don't believe in chaos, upload it here. I promise, it won't work as well.

How does this work?

Notice that if you try encoding the same image multiple times, the result changes. The mask overlay has a random element to it, so the decoder is clearly not just recreating the mask exactly.

So, how does this work? The mask comes from a time series of a Lorenz system. The decoder is another Lorenz system set to synchronise with the original one. The decoder tries to recreate the mask, then subtracts its reconstruction. Of course, it isn't a perfect reconstruction. The data it is based on is a sum of chaotic data and image data. The theory is that the chaotic system doesn't understand image data at all, so it just ignores it, and only reconstructs the chaotic part. This was inspired by work from Steven Strogatz, as explained here.

There are two big problems with this scheme. Strogatz originally used it on sound, and he (well, his student Kevin Cuomo) built the Lorenz system as an analog circuit. I tried the scheme with image data and, at first, failed utterly. The problem is that image data is purely positive (there can't be negative brightness), while sound is an oscillation. It turns out that the Lorenz system receiver can pick up a shift much better than oscillations. I solved this by multiplying the image data with a parity bit: alternating plus or minus one.

So, is this useful at all?

Probably not for sending actual secrets, especially because you can't easily give this system a key as a parameter, so anybody can decode your image on this site. (There might be a way, though...) Instead, this is a demonstration of a kind of signal processing. With ordinary periodic signals, it's easy to build a device that picks out a particular frequency and ignores everythng else: that's how radios work. Maybe there is some way to encode information into the chaotic mask. Then, this mask can be synchronised with.

Already in this demonstration, I fiddle with the mask a little bit, changing it by a tiny random factor to make it unique each time. It is known that chaotic systems can be stabilised to periodic orbits with the smallest nudges, if applied at the right time in the right way. Perhaps you could also encode zeros and ones somewhere in the number of orbit the system makes on either side before flipping.

And why would you want to do that? Maybe (I haven't tested this) such a system would be more resistant to periodic noises. Maybe (I haven't tested this) you could run multiple chaotic systems at once, with different parameters, and they would not interact. I doubt it, though. I haven't yet had a problem that could be solved with this, but perhaps one day I will, and then I'll try this. Till then, have this toy!
